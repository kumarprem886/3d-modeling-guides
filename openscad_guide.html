OpenSCAD — Complete Guide

Code-Based Parametric 3D Modeling

OpenSCAD is different: you WRITE CODE to make 3D models, not draw them. Every shape, position, and dimension is defined in a text script — making it the ultimate parametric tool.

There is no transform gizmo in OpenSCAD — all positioning is done through code using translate(), rotate(), and scale(). This gives you exact, reproducible control.
OpenSCAD has no undo history for geometry changes — use your code editor's undo (Ctrl+Z) and save frequently. Consider using version control (Git) for complex projects.

3D Primitives

cube([x, y, z]);          // rectangular box
sphere(r=5);              // sphere
cylinder(h=10, r=5);      // cylinder
cylinder(h=10, r1=5, r2=0); // cone
polyhedron(points=[], faces=[]); // custom mesh

// center=true places shape at origin
cube([10,10,10], center=true);

Smoothness Control ($fn)

// $fn controls curve resolution
sphere(r=5, $fn=16);  // fast draft quality
sphere(r=5, $fn=64);  // good balance
sphere(r=5, $fn=128); // smooth, slow render

// Set globally at top of file:
$fn = 64; // applies to all curves
Use $fn=16 while designing for fast previews. Change to $fn=128 only for the final F6 render before export — high $fn makes rendering very slow.

Boolean Operations

union() {          // combine shapes together
  cube(10);
  sphere(7);
}

difference() {     // first shape minus the rest
  cube(10);
  cylinder(h=15, r=3);
}

intersection() {   // keep only shared volume
  cube(10);
  sphere(7);
}

Transforms

translate([x,y,z]) shape;   // move
rotate([x,y,z]) shape;      // rotate degrees
scale([x,y,z]) shape;       // scale
mirror([1,0,0]) shape;     // flip on X axis

linear_extrude(height=10) circle(5); // 2D → 3D
rotate_extrude() polygon;  // revolve 2D profile

hull() { shapes }           // convex hull
minkowski() { shapes }     // rounded shapes trick
Minkowski trick for rounded boxes: minkowski() { cube([8,8,8]); sphere(r=2); } — creates a box with rounded corners without complex math.

Variables & Modules

// Variables — parametric design
wall = 3;
height = 20;
diameter = 15;
cylinder(h=height, r=diameter/2);

// Modules — reusable functions
module roundedBox(size, radius) {
  minkowski() {
    cube(size);
    sphere(r=radius);
  }
}
roundedBox([10,10,10], 1);

Loops & Conditionals

// For loop — repeat geometry
for(i = [0:5]) {
  translate([i*15, 0, 0]) sphere(5);
}

// Conditional logic
if (wall > 2) {
  cube(10);
} else {
  sphere(5);
}

// Include external files
include <helpers.scad>
use <library.scad>
Define all design variables at the top of your file. This makes your design parametric — change one number to update the entire model instantly on F5 preview.

2D Primitives

square([x,y]);       // 2D rectangle
circle(r=5);         // 2D circle
polygon(points=[[0,0],[10,0],[5,10]]); // triangle

// Offset 2D shape (expand / contract)
offset(r=2) square(10);  // rounded corners
offset(delta=-1) circle(10); // shrink

Text & Import

// 3D text (use linear_extrude to make solid)
linear_extrude(3)
  text("Hello", size=10, font="Arial");

// Import SVG as flat 2D shape
linear_extrude(2) import("logo.svg");

// Import DXF
import("drawing.dxf");
2D shapes in OpenSCAD are used as profiles for linear_extrude() and rotate_extrude(). Draw complex 2D profiles in Inkscape, export as SVG, then import and extrude in OpenSCAD.

Coding Tips

Use echo(wall=wall); to print variable values to the console for debugging — essential for tracking down calculation errors.
Add /* [Group Name] */ comments above variables to create interactive Customizer UI sliders (Design → Customizer). Great for sharing parametric models.
Comments: // single line or /* multi-line */. Use comments liberally to document what each variable and module does.
Use the OpenSCAD Web IDE at openscad.cloud for quick experiments without installing anything.

Workflow: Parametric 3D Printing

  1. Define all design variables at the top of your .scad file: wall thickness, height, diameter, tolerance, etc. These become your parametric controls.
  2. Build basic shapes using primitives and boolean operations. Start with the outer envelope, then subtract holes and internal features using difference().
  3. Create module definitions for repeated elements (mounting legs, bolt holes, snap clips). Call them with for loops for arrays.
  4. Test with F5 preview. Adjust variables at the top and re-preview to see live changes. Use echo() to verify computed values.
  5. Once satisfied, press F6 for a full render (may take minutes for complex designs with high $fn values). Wait for "Rendering done" in status bar.
  6. Export: File → Export → Export as STL (or press F7). The STL is now ready for your slicer.
  7. Import the STL into your slicer (PrusaSlicer, Cura, Bambu Studio) → configure print settings → slice → send to printer.
OpenSCAD is completely free and open-source, runs on Windows, macOS, and Linux. It's the favorite tool of programmers who want precise, reproducible, version-controllable 3D designs.
