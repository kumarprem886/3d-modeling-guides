⚙️ What is G-code?

G-code is the universal language of CNC machines and 3D printers. Every layer, every move, every temperature change is a G-code command.

How It Works

Your slicer (Bambu Studio, PrusaSlicer, Cura) takes your 3D model and converts it into thousands of G-code lines. Your printer reads these line by line and executes each instruction precisely.

📐 G Commands
Geometric / movement commands. Tell the machine where to go and how fast. G0, G1, G28, G29…
🔧 M Commands
Machine commands. Control hardware states — temperatures, fans, motors, LEDs. M104, M106, M420…
💬 Comments
Lines starting with ; are ignored by the printer. Slicers use them to annotate layer changes, settings, timestamps.
🔢 Coordinates
X, Y, Z = axes. E = extruder. F = feedrate (mm/min). All distances are in millimetres.

Anatomy of a G-code Line

; This is a comment — ignored by the printer G1 X100.5 Y75.0 Z0.2 E1.245 F3000 ; G1 = linear move ; X/Y/Z = destination in mm ; E = total filament extruded (mm) ; F = feedrate in mm/min (3000 = 50 mm/s)

Units & Conventions

  • All distances: millimetres (G21 sets this, always default)
  • Feedrate (F): mm/min — divide by 60 to get mm/s
  • Temperature: °C
  • Fan speed: 0–255 (Marlin) or 0–1.0 percentage (Klipper)
  • Extrusion (E): cumulative mm since start, or per-move delta in relative mode

📄 G-code File Structure

A well-formed G-code file has a predictable structure. Understanding it lets you edit files manually and write powerful start/end scripts.

Full File Anatomy

; ────────────────────────────────────────── ; HEADER — slicer metadata (all comments) ; Generated by PrusaSlicer 2.8.0 ; Layer height: 0.2mm ; Filament: PLA, 215°C / 60°C bed ; ────────────────────────────────────────── ; ── INIT — machine units & absolute mode G21 ; millimetres G90 ; absolute positioning M82 ; extruder absolute ; ── START SCRIPT (user-defined in slicer) M104 S215 ; set hotend (no wait) M140 S60 ; set bed (no wait) G28 ; home all axes M190 S60 ; wait for bed temp M109 S215 ; wait for hotend temp G29 ; auto bed leveling ; ── PRINT BODY (slicer generated) ; Layer 1 G0 F9000 X10 Y10 Z0.2 G1 F1800 X200 Y10 E5.432 ; ... thousands more lines ... ; ── END SCRIPT M104 S0 ; hotend off M140 S0 ; bed off M106 S0 ; fan off G91 ; relative mode G1 Z10 ; lift nozzle 10mm G28 X Y ; park printhead M84 ; disable steppers

Slicers embed special comments as hooks — some firmware reads these:

; TYPE:WALL-OUTERCura feature type (walls, infill, support…)

Editing G-code Manually

✅ Safe to edit: Start/end scripts, temperatures, fan speed values, travel feedrate (F on G0 lines), comments.
⚠️ Be careful editing: E values (affect flow), Z values (crash risk), G29 removed without compensation loaded.
🚫 Never edit: X/Y coordinates of the print body unless you know exactly what you're doing — dimensional errors compound over thousands of lines.

🎯 Movement Commands (G-codes)

Movement commands control where the printhead goes, how fast, and in what coordinate system.

Core Movement

CommandParametersDescription
G0X Y Z FRapid/travel move — no extrusion. Printer moves as fast as it safely can. F sets max speed.
G1X Y Z E FLinear move with extrusion. The main printing command — moves and pushes filament simultaneously.
G2X Y I J F EClockwise arc move. I/J = arc centre offset from current position.
G3X Y I J F ECounter-clockwise arc move. Rarely used in FDM slicers.
G4P (ms) or S (sec)Dwell/pause. G4 P500 = wait 500ms. G4 S2 = wait 2 seconds.
; Travel to start of next line (no extrusion) G0 F9000 X50 Y50 ; Print a line at 50mm/s (F3000 = 3000mm/min ÷ 60 = 50mm/s) G1 F3000 X150 Y50 E8.250 ; Dwell 1 second (let bridge cool before continuing) G4 S1

Homing

CommandParametersDescription
G28X Y Z (optional)Home axes. G28 = all. G28 X Y = X and Y only. G28 Z = Z only.
G28.2Home with sensorless homing (Klipper). Klipper
⚠️ Always G28 before G29 or any probing. The probe sequence assumes the machine has a known reference point.

Coordinate Modes

CommandDescription
G90Absolute positioning — X/Y/Z coordinates are from machine origin. Default for most prints.
G91Relative positioning — X/Y/Z are offsets from current position. Useful in end scripts (G1 Z10 = lift 10mm).
G92Set current position — defines where the machine "thinks" it is without moving. G92 E0 resets extruder counter.
; Reset extruder position to 0 (very common in start scripts) G92 E0 ; End script: switch to relative, lift nozzle, park, restore absolute G91 G1 Z10 F600 ; lift 10mm relative G90 G0 X0 Y220 ; park at back

Units & Feedrate Tips

G20 / G21
G20 = inches, G21 = mm. Always use G21. Your slicer already sets this.
Feedrate Math
F is mm/min. To print at 60 mm/s → F3600. To travel at 150 mm/s → F9000.
F is Sticky
Once set, F stays until changed. You don't need to repeat it on every line — only when the speed changes.
G0 vs G1
On most 3D printer firmware, G0 and G1 are identical — both do linear moves. G0 is convention for travel (no E).

🌡️ Temperature & Fan Commands (M-codes)

M-codes control machine hardware state — heaters, fans, motors, LEDs, and more.

Hotend Temperature

CommandParametersDescription
M104S[temp] T[extruder]Set hotend temperature — non-blocking. Continues to next line immediately.
M109S[temp] R[temp] T[extruder]Set hotend + wait until reached. Use in start script before printing. R = wait for exact temp (cool OR heat).
M104 S0—Turn hotend OFF. Always in end script.
; Preheat while homing (non-blocking — saves time) M104 S215 ; start heating, don't wait G28 ; home while heating M109 S215 ; now wait until 215°C

Bed Temperature

CommandParametersDescription
M140S[temp]Set bed temperature — non-blocking.
M190S[temp] R[temp]Set bed + wait. Use after homing, before M109.
M140 S0—Turn bed heater OFF.
💡 Optimal order: M104 (preheat hotend) → M140 (preheat bed) → G28 (home) → M190 (wait bed) → M109 (wait hotend). This saves 1–2 minutes on every print.

Fan Control

CommandParametersDescription
M106S[0-255] P[fan index]Set part cooling fan speed. S255 = 100%. S128 = ~50%.
M107—Part cooling fan OFF. Same as M106 S0.
; Turn fan off for first layer (better adhesion) M107 ; After layer 3, turn fan to 100% M106 S255 ; Bridge layer — reduce fan to 30% to prevent warping M106 S77

Chamber & Other Heaters

M191 S[temp]MarlinSet chamber + wait
M116BothWait for all heaters to reach target

🔩 Extruder & Flow Control

The E axis controls how much filament is pushed. Getting this right is fundamental to print quality.

Absolute vs Relative Extrusion

CommandDescription
M82Absolute extrusion — E values are total mm from start (or last G92 E0). Default in most slicers.
M83Relative extrusion — each E value is distance for that move only. Simpler math, used by some slicers.
G92 E0Reset extruder position counter to 0. Common in start scripts and after retracts.
; Absolute mode — E keeps growing M82 G92 E0 G1 X100 E5.0 ; extruded 5mm total G1 X200 E10.5 ; extruded 5.5mm more (total 10.5) ; Relative mode — E is per-move delta M83 G1 X100 E5.0 ; extrude 5mm G1 X200 E5.5 ; extrude 5.5mm more

Retraction

Retraction pulls filament back before a travel move to prevent stringing. Slicer handles this automatically, but you can tune or script it.

; Retract 0.8mm at 45mm/s (F2700) G1 F2700 E9.7 ; was at E10.5, now pull back 0.8mm ; Travel move G0 F9000 X50 Y80 ; Un-retract (prime) before next print move G1 F2700 E10.5 ; push back the 0.8mm

Retraction by Material

MaterialDirect Drive (mm)Bowden (mm)Speed (mm/s)
PLA0.5–1.03.5–6.025–45
PETG0.5–1.54.0–7.020–35
ABS0.5–1.03.0–5.025–40
TPU0.5–2.0Not recommended15–25

Flow Rate Override

CommandDescription
M221 S[%]Set flow rate multiplier. M221 S95 = 95% flow (underextrusion). Default 100.
M220 S[%]Set speed multiplier. M220 S150 = 150% speed. Applies globally.
💡 E-step Calibration — The most important extruder setting. Mark 100mm on filament, command G1 E100 F100, measure what actually moved. If 98mm moved: new steps = old steps × (100 / 98). Do this first on any new printer.

Extruder Motor

CommandDescription
M84Disable all steppers (motors go limp). Use in end script.
M84 EDisable only extruder motor.
M18Same as M84 — disable steppers.
M17Enable all steppers.

📏 Bed Leveling & Probing

A level bed is the single biggest factor in first-layer success. G-code gives you full control over the probing process.

Homing & Z-offset

CommandDescription
G28Home all axes. Must run before any probing.
G28 ZHome Z only (re-trigger Z endstop/probe).
M851 Z[-offset]Set Z probe offset. Negative = nozzle is below probe. Marlin
M206 Z[value]Set home offset (alternative Z offset method). Marlin
; Set Z probe offset to -1.25mm (probe is 1.25mm above nozzle) M851 Z-1.25 M500 ; save to EEPROM

Auto Bed Leveling (ABL)

CommandParametersDescription
G29variesRun ABL sequence. Probes multiple points and creates a mesh. Behavior depends on firmware config.
G29 A—Activate UBL mesh Marlin
G29 S1—Save UBL mesh to slot 1 Marlin
G29 L1—Load UBL mesh from slot 1 Marlin
M420 S1—Enable bed leveling compensation. Use in start script after G28 if you saved mesh. Marlin
M420 S0—Disable bed leveling compensation temporarily.
BED_MESH_CALIBRATE—Run bed mesh (equivalent of G29) Klipper
BED_MESH_PROFILE LOAD=default—Load saved mesh Klipper
💡 Smart start script pattern (Marlin): Run G29 only on first print of the day. All subsequent prints use M420 S1 to load the saved mesh without re-probing — saves 3–5 minutes per print.
; Efficient start script — load saved mesh instead of probing every time G28 ; home all M420 S1 ; enable saved mesh ; To reprobe instead: replace M420 S1 with G29

Manual Tramming

CommandDescription
G35Assisted tramming — probes corners and tells you how many turns each screw needs. Marlin
SCREWS_TILT_CALCULATEKlipper equivalent — reports clockwise/CCW turns per screw. Klipper

Probe Commands

CommandDescription

📝 Start & End G-code Scripts

The start and end scripts run before and after every print. Getting these right saves time and prevents failures.

Universal Start Script (PLA / Marlin)

; ─── START GCODE ─────────────────────────────── G21 ; mm units G90 ; absolute positioning M82 ; absolute extrusion ; Preheat while homing (parallel = time saving) M104 S215 ; start hotend (don't wait) M140 S60 ; start bed (don't wait) G28 ; home all axes M190 S60 ; wait for bed 60°C M109 S215 ; wait for hotend 215°C M420 S1 ; load saved mesh (or G29 to re-probe) ; Purge line along left edge G92 E0 G0 F9000 X1 Y20 Z0.3 G1 F1500 X1 Y200 E16 ; draw purge line G1 F2700 E15.2 ; retract G92 E0 ; reset E G0 F9000 X5 Y20 ; wipe off blob

Universal End Script

; ─── END GCODE ───────────────────────────────── M104 S0 ; hotend off M140 S0 ; bed off M106 S0 ; fan off G91 ; relative mode G1 Z10 F600 ; lift nozzle 10mm G1 E-2 F2700 ; retract filament 2mm G90 ; back to absolute G0 X0 Y235 F9000 ; park at front-left / present print M84 ; disable steppers

Klipper Start Script

; Klipper uses macros in printer.cfg — call them from slicer start gcode PRINT_START EXTRUDER=215 BED=60 ; Your printer.cfg macro handles homing, heating, QGL, mesh, purge ; [gcode_macro PRINT_START] ; gcode: ; {% set EXTRUDER = params.EXTRUDER|default(200)|int %} ; {% set BED = params.BED|default(60)|int %} ; M104 S{EXTRUDER} ; M140 S{BED} ; G28 ; BED_MESH_CALIBRATE ; M109 S{EXTRUDER} ; M190 S{BED}

🚀 Advanced Tuning Commands

Beyond basic printing — commands for calibration, compensation, and firmware configuration.

EEPROM (Persistent Storage)

CommandDescription
M500Save settings to EEPROM. Run after any calibration change to make it permanent across power cycles.
M501Load settings from EEPROM (restore saved values). Run after accidental changes.
M502Factory reset — load firmware defaults. Run M500 after if you want to save them.
M503Print current settings. Shows all EEPROM values — steps/mm, PID, offsets, etc.
💡 Always run M503 first before calibrating a new/used printer. Know your baseline before changing anything.

Steps per mm

CommandExampleDescription
M92M92 X80 Y80 Z400 E415Set steps/mm for each axis. Critical for dimensional accuracy.
; Read current steps M503 ; Output: M92 X80.00 Y80.00 Z400.00 E415.00 ; Example: extruder calibration — commanded 100mm, got 96mm ; New E steps = 415 × (100/96) = 432.29 M92 E432.29 M500 ; save

M304 P[p] I[i] D[d]Set bed PID values manually.

Linear Advance / Pressure Advance

Compensates for filament pressure lag — improves corners and sharp features dramatically.

CommandFirmwareDescription
M900 K[value]MarlinSet Linear Advance factor K. Typical PLA: 0.05–0.2. Bowden: 0.5–1.5.
SET_PRESSURE_ADVANCE ADVANCE=[value]KlipperSet pressure advance. Typical: 0.04–0.08 for direct drive.
SET_PRESSURE_ADVANCE SMOOTH_TIME=0.04KlipperSmooth time for PA. Default 0.04s is good for most printers.
; Marlin — enable and set Linear Advance for PLA M900 K0.08 M500 ; Klipper — in printer.cfg ; [extruder] ; pressure_advance: 0.06 ; pressure_advance_smooth_time: 0.040

Input Shaping (Resonance Compensation)

Eliminates ghosting/ringing artifacts caused by printer vibration.

CommandFirmwareDescription
M593 F[freq] D[damping]MarlinSet ZV input shaper. Measure resonance frequency with accelerometer or test print.
SET_INPUT_SHAPER SHAPER_TYPE=mzv SHAPER_FREQ_X=50KlipperSet shaper type and frequency.
SHAPER_CALIBRATEKlipperAuto-calibrate with ADXL345 accelerometer. Most accurate method.

Acceleration & Jerk

CommandDescription
M201 X[acc] Y[acc]Set max acceleration per axis (mm/s²). Marlin default: 500–1000.
M204 P[print] T[travel]Set print acceleration and travel acceleration.
M205 X[jerk] Y[jerk]Set jerk (Marlin) — max instantaneous velocity change. Lower = smoother, higher = faster corners.

Writing Klipper Macros

; In printer.cfg — a full PRINT_START macro [gcode_macro PRINT_START] gcode: {% set EXTRUDER = params.EXTRUDER|default(200)|int %} {% set BED = params.BED|default(60)|int %} M104 S{EXTRUDER} # preheat nozzle (no wait) M140 S{BED} # preheat bed (no wait) G28 # home all QUAD_GANTRY_LEVEL # level gantry (Voron) BED_MESH_CALIBRATE # probe mesh M190 S{BED} # wait bed M109 S{EXTRUDER} # wait nozzle G92 E0 # purge line G0 X5 Y5 Z0.3 F9000 G1 X5 Y200 E18 F1500 G92 E0

Conditional Logic in Macros

; Jinja2 templating in Klipper macros [gcode_macro HEAT_SOAK] gcode: {% set TEMP = params.TEMP|default(110)|int %} {% if TEMP > 100 %} M141 S{TEMP} # heat chamber M191 S{TEMP} # wait for chamber {% endif %} M190 S{TEMP} # bed to chamber temp

Useful Klipper Extras (moonraker / klicky / etc.)

🔃 Object Cancel
Slicer annotates objects → Klipper can cancel individual objects mid-print. Enable: EXCLUDE_OBJECT_START NAME=...
📊 ADXL345
Accelerometer wired to MCU → SHAPER_CALIBRATE auto-tunes input shaping. Eliminates ringing permanently.
🔄 Endless Spool
MMU/filament sensor macros let Klipper auto-switch spools when one runs out.
🌡️ Thermal Expansion
SET_GCODE_OFFSET Z=0.01 live-adjust Z without stopping. Great for first-layer tuning.
